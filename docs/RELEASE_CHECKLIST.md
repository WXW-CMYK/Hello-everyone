# Smart Brochure Generator 发布前检查清单

本文档用于每次发布前的人工检查。建议每个平台分别记录一次结果。

## 基础信息

- [ ] 确认版本号已更新。
- [ ] 确认 `package-lock.json` 与依赖版本一致。
- [ ] 确认 README 中的运行、打包和测试说明仍然有效。
- [ ] 确认已记录本次发布的主要变更。

## Build

- [ ] 执行 `npm ci`。
- [ ] 执行 `npm run build`。
- [ ] 确认 TypeScript、main、preload 和 renderer 构建通过。
- [ ] 确认没有新增阻塞级别 warning。

## macOS DMG

- [ ] 执行 `npm run dist:mac`。
- [ ] 确认 `release/` 中生成 `.dmg`。
- [ ] 打开 DMG，确认 app 名称正确。
- [ ] 确认 app 图标正确。
- [ ] 打开 app，确认可以正常启动。
- [ ] 记录未签名 Gatekeeper 提示情况。

## Windows EXE

- [ ] 通过 GitHub Actions 或 Windows 本机执行 `npm run dist:win`。
- [ ] 确认 `release/` 中生成 `.exe` 安装包。
- [ ] 在 Windows 真机或干净 VM 中运行安装包。
- [ ] 确认安装完成。
- [ ] 确认开始菜单或桌面快捷方式图标正确。
- [ ] 打开 app，确认可以正常启动。
- [ ] 记录 SmartScreen、Defender 或杀毒软件提示情况。

## 图标

- [ ] 确认 `build/icon.png` 存在。
- [ ] 确认 `build/icon.icns` 存在。
- [ ] 确认 `build/icon.ico` 存在。
- [ ] 确认 macOS app 使用 `build/icon.icns`。
- [ ] 确认 Windows 安装包使用 `build/icon.ico`。

## API 设置

- [ ] 进入设置页。
- [ ] 保存 API Key。
- [ ] 保存 Endpoint。
- [ ] 保存 Model。
- [ ] 切换 mock fallback 并保存。
- [ ] 关闭并重新打开 app，确认设置保留。
- [ ] 验证 API Key 默认隐藏，显示/隐藏切换可用。

## 创建项目

- [ ] 创建新宣传册项目。
- [ ] 填写产品基础信息。
- [ ] 使用 mock fallback 生成 AI 内容。
- [ ] 如本次发布需要，使用真实 API Key 生成 AI 内容。
- [ ] 进入项目列表，确认项目显示正确。

## 编辑器保存

- [ ] 点击项目名称进入编辑器。
- [ ] 修改封面标题和副标题。
- [ ] 修改产品介绍。
- [ ] 修改核心卖点、痛点、解决方案、应用场景、规格、FAQ 和 CTA。
- [ ] 切换 4 套模板。
- [ ] 上传 Logo。
- [ ] 上传产品图片。
- [ ] 点击保存。
- [ ] 确认保存状态显示为已保存。
- [ ] 返回项目列表后再次进入，确认内容保留。

## PDF 导出

- [ ] 设置 PDF 文件名前缀。
- [ ] 在编辑器点击导出 PDF。
- [ ] 确认保存对话框默认文件名正确。
- [ ] 导出到本地目录。
- [ ] 确认导出成功提示显示。
- [ ] 点击打开文件所在位置。
- [ ] 打开 PDF。
- [ ] 确认 PDF 包含模板、Logo、产品图和编辑后的文案。
- [ ] 确认多页内容没有明显截断、重叠或溢出。

## 重启持久化

- [ ] 完成设置保存、项目创建、图片上传和 PDF 导出后关闭 app。
- [ ] 重新打开 app。
- [ ] 确认设置保留。
- [ ] 确认项目列表保留。
- [ ] 确认编辑器内容保留。
- [ ] 确认模板选择保留。
- [ ] 确认 Logo 和产品图片仍然显示。
